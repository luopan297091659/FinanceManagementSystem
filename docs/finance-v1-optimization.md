# 入出金管理 V1 优化说明

## 技术范围

- 前端：Vue 3
- 后端：NestJS + Prisma + PostgreSQL
- 不再维护 Python 服务
- OCR 与 AI 解析归入 V2，不作为 V1 入出金闭环的前置依赖

## V1 已补齐的基础能力

1. 入出金台账增加文件金额、手续费、统计金额、摘要原文、证据页码、计入状态与确认状态。
2. 增加银行名义别名、PDF 页覆盖、业主结算、异常预警和阈值配置的数据模型，为对账、重复件控制和异常处理预留持久化基础。
3. 支持流水创建、编辑、删除；录入时复用既有费用项，不再因每笔流水自动新增费用项。
4. 前端支持日语/中文切换，默认日语；出入金状态使用日文业务语义展示。
5. 合同预留动态收款周期、提醒天数、下次应收日、费用承担方及三项其他月度费用，支持后续逾期和业主结算规则。

## 建议的 V1 后续实现顺序

1. 合同管理页补充收款周期、下次应收日、提醒日、费用承担方字段。
2. 建立银行流水导入和人工匹配页，维护银行名义别名并生成对账结果。
3. 实现异常预警列表：未匹配入金、逾期、金额差异、重复件、页码缺失和业主汇款异常。
4. 实现业主结算草稿、审核、汇款状态和审计日志。
5. 增加角色权限与关键状态变更审计。

## V2 范围

- OCR 文档识别、PDF 自动分页覆盖和结构化字段抽取。
- AI 辅助归类、银行名义模糊匹配、异常原因建议与知识库问答。
- 外部通知渠道和自动化工作流。
